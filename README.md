# GaryMariner.com — AI-Augmented Brand System

## Overview

This project is the official website for Gary Mariner, an AI Systems Architect for Business Operations.

More than a personal brand site, this repository represents a **structured, AI-augmented approach to building premium digital experiences**.

The goal is to demonstrate how modern AI-assisted systems can:

- reduce development overhead  
- improve speed of execution  
- maintain high-quality output  
- create scalable, structured digital infrastructure  

---

## Project Purpose

GaryMariner.com serves as:

## System Architecture

This project is built using a documentation-first approach.

All decisions are guided by structured `.md` files located in the `/docs` directory.

### Core Documents

- `brand-strategy.md` → positioning, audience, and identity  
- `site-architecture.md` → page structure and user flow  
- `homepage-wireframe.md` → layout and experience design  
- `homepage-copy.md` → messaging and content  
- `design-system.md` → visual rules and UI standards  
- `case-study-digital-harmony.md` → primary proof and credibility  
- `CLAUDE.md` → system instructions for AI-assisted development  

---

## Development Approach

This project follows an **AI-Augmented development model**:

1. Define strategy through structured documentation  
2. Use AI (Claude) to interpret and execute based on those documents  
3. Build modular, reusable components  
4. Refine through iterative feedback  
5. Maintain clarity, structure, and performance  

---

## Technology Stack

- **Framework:** Next.js  
- **Styling:** Tailwind CSS  
- **Hosting:** Vercel  
- **Forms:** Lightweight form handling (Resend / server actions)  
- **Scheduling:** Cal.com or similar  
- **Analytics:** Plausible or Google Analytics  

The stack is intentionally minimal to reduce complexity and overhead.
